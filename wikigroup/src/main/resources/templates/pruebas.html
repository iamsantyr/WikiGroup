Pruebas del Proyecto

(Textos realizados por el equipo) — Esta sección documenta los casos de prueba de frontend y backend.

1. Pruebas Frontend (Formulario Contáctenos)

ID Escenario Pasos Resultado Esperado Resultado
PF-01 Contadores de caracteres Abrir /contacto, escribir en Nombres/Apellidos/Descripción. Los contadores muestran escritos/máximo y no permite exceder el máximo. OK / FALLA
PF-02 Normalización a MAYÚSCULAS (sin tildes/ñ) Escribir “José Ñuñez”. Se muestra “JOSE NUNEZ”. OK / FALLA
PF-03 Correo sin espacios y con @ y punto Escribir “mail sin espacios @gmail .com”. Marca en rojo y muestra mensaje “Correo inválido…”. OK / FALLA
PF-04 Semestre 0–16 Ingresar -1 o 17. Marca en rojo y bloquea envío. OK / FALLA
PF-05 Bloqueo de envío con campos inválidos Dejar vacíos o con error y presionar Enviar. Aparece alerta “Revisa los campos en rojo.” y no se envía. OK / FALLA

2. Pruebas Backend (Validaciones y Persistencia)

ID Caso Entrada Resultado Esperado Resultado
PB-01 Correo inválido correo = "SIN@PUNTO" Bean Validation rechaza con mensaje de patrón. OK / FALLA
PB-02 Semestre fuera de rango semestre = -1 / 17 Rechazo por @Min/@Max. OK / FALLA
PB-03 Persistencia exitosa Todos válidos Registro aparece en tabla contacto. OK / FALLA
