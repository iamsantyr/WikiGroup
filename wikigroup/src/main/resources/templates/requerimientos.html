Requerimientos Funcionales

ID Descripción Criticidad Criterios de Aceptación
RF-01 El sistema deberá permitir registrar mensajes de contacto. Alta Dado un usuario, cuando completa el formulario válido, entonces el mensaje se almacena en DB.
RF-02 El sistema listará los mensajes creados ordenados por fecha. Media Dado mensajes existentes, cuando visito /mensajes, entonces se visualizan en tabla.
RF-03 El sistema permitirá eliminar un mensaje por su identificador. Media Dado un mensaje, cuando hago clic en “Eliminar”, entonces el registro desaparece y se notifica.

Requerimientos No Funcionales

ID Categoría Descripción Métrica
RNF-01 Usabilidad El formulario mostrará validaciones en cliente en tiempo real. Contadores de caracteres, mensajes de error y bloqueo si hay campos inválidos.
RNF-02 Compatibilidad La wiki deberá visualizarse correctamente en Chrome, Safari, Edge y Firefox. Pruebas manuales en los cuatro navegadores.
RNF-03 Mantenibilidad El menú será un fragmento Thymeleaf reutilizable. Agregar un ítem sin editar otras vistas.

Suposiciones y Dependencias

  • Base de datos PostgreSQL accesible localmente.
  • Despliegue final vía K8s + Jenkins según sesión de clase.
